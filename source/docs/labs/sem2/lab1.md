# Лабораторная работа №1: Создание и развертывание статического сайта на MkDocs

## Ссылка на GitHub: [Репозиторий с кодом](https://github.com/jUmb0-juiceee/ПОКА_НЕТ_ССЫЛКИ)

## Описание задачи

В этой работе нужно было создать статический сайт-портфолио с использованием генератора MkDocs и опубликовать его на GitHub Pages. Задача включала настройку виртуального окружения, установку MkDocs, выбор и подключение темы Material, настройку конфигурационного файла mkdocs.yml, создание структуры страниц и публикацию сайта на GitHub Pages. В результате должен получиться сайт, доступный по адресу username.github.io, с главной страницей, страницей "Об авторе" и разделом лабораторных работ.

## Ход работы

**1. Создание репозитория и настройка GitHub Pages.** Создал публичный репозиторий на GitHub. В настройках репозитория выбрал источник публикации: ветка main, папка /docs.

**2. Клонирование и настройка окружения.** Склонировал репозиторий на локальный компьютер. Создал виртуальное окружение через python -m venv .venv и активировал его. Установил MkDocs и тему Material через pip.

**3. Настройка .gitignore.** Создал файл .gitignore, в который добавил виртуальное окружение (.venv/), служебные файлы Python (__pycache__/, *.pyc) и папку сборки (site/).

**4. Создание сайта.** Выполнил команду mkdocs new source, которая создала базовую структуру сайта.

**5. Настройка mkdocs.yml.** Настроил конфигурационный файл: указал название сайта, тему Material с переключением светлой/тёмной темы, добавил функции навигации (navigation.tabs, navigation.sections, content.code.copy). Прописал структуру меню через nav: главная, раздел лабораторных работ с семестрами, страница об авторе.

**6. Создание страниц.** Создал Markdown-файлы для главной страницы, страницы об авторе и страниц для каждой лабораторной работы (1-8). Для каждой страницы с лабораторной работой использовал единый шаблон: ссылка на код, описание задачи, ход работы, выводы.

**7. Локальный запуск.** Запустил локальный сервер через mkdocs serve, проверил работу сайта в браузере.

**8. Сборка сайта.** Выполнил сборку сайта в папку /docs корня репозитория через mkdocs build -d ../docs.

**9. Публикация.** Закоммитил изменения и запушил их в удалённый репозиторий. GitHub Pages автоматически развернул сайт.

## Выводы

В ходе работы я освоил создание статического сайта с помощью MkDocs, настройку темы Material с переключением цветовых схем, организацию структуры документации через mkdocs.yml и nav. Научился работать с виртуальным окружением, собирать сайт и публиковать его на GitHub Pages. Сайт успешно развернут и доступен по ссылке. Этот опыт пригодится для создания документации к любым проектам.
