## Описание задачи

Данные не всегда лежат в готовых csv-файлах. Часто их нужно собирать с веб-страниц самостоятельно. Это называется веб-скрапингом. В этой работе мы парсили новости с сайта ИТМО — https://news.itmo.ru/ru. Задача была собрать данные по двум поисковым запросам ("нейротехнологии" и "нейротехнологии и программирование"), а потом для каждой новости спарсить детальную информацию: идентификатор, название, дату, количество просмотров, текст и теги. Всё это нужно было сохранить в CSV.

## Ход работы

**1. Установка библиотек.** Установил библиотеки: requests для отправки HTTP-запросов и BeautifulSoup для парсинга HTML.

**2. Формирование запросов.** Задал домен сайта и URL для поиска. Сформировал поисковые запросы — "нейротехнологии" и "нейротехнологии и программирование".

**3. Сбор ссылок на новости.** Для каждого запроса отправил GET-запрос, получил HTML-страницу с результатами поиска. В BeautifulSoup нашёл все элементы с классом weeklyevent — внутри них лежали ссылки на новости. Прошёлся по каждой новости и сохранил заголовок и полный URL. Собрал 20 ссылок с двух страниц поиска.

**4. Парсинг детальной информации.** Дальше по заданию нужно было спарсить детальные данные для каждой новости. Я пошёл по ссылкам, для каждой страницы нашёл через BeautifulSoup: заголовок (тег h1), дату (тег time), просмотры (искал внутри тега time класс icon eye), текст (div с классом content) и теги (ul с классом tags, откуда вытащил все ссылки с названиями тегов). Все поля сохранил в список словарей.

**5. Особенности парсинга main_news.** Столкнулся с тем, что у обычных новостей и у main_news разная структура. Для main_news вообще не было заголовков на страницах-списках, поэтому пришлось сначала собирать ссылки на новости с каждой из 127 страниц main_news, а потом уже по ним собирать детальные данные. В итоге получил 81 уникальную ссылку на новости с первых 9 страниц (процесс был долгим, поэтому взял только 9 страниц).

**6. Поиск просмотров.** Для парсинга просмотров пришлось найти, где они лежат на странице. Оказалось, что они внутри тега time, в span с классом icon eye — этот класс не находится через стандартный поиск, только через уточнение внутри родительского элемента.

**7. Парсинг тегов.** Теги на страницах лежат в div с классом label — после слова "Теги" идёт список ul с классом tags. Я вытащил все ссылки из этого списка и склеил названия через запятую. В процессе проверки структуры тегов нашёл, что они лежат внутри group, где есть label с текстом "Теги" и content с ul.tags. Внутри каждого li была ссылка a с названием тега.

**8. Сохранение данных.** После того как парсинг всех полей заработал, сохранил данные в CSV через стандартный модуль csv. В файл записались все поля: id, title, date, url, views, text, tags.

## Выводы

Скрапинг — это не всегда просто. У разных страниц на одном сайте может быть разная вёрстка, и это надо учитывать. Для страниц main_news пришлось сначала собирать ссылки на новости, а потом уже парсить каждую страницу по отдельности. Просмотры оказались не в отдельном блоке, а внутри тега time, и найти их удалось только через родительский элемент. Теги на странице есть, они лежат в ul с классом tags. Самый сложный момент в скрапинге — найти правильные селекторы для данных. Иногда они лежат не там, где ожидаешь, и приходится перебирать разные варианты. В итоге удалось собрать 45 записей о новостях ИТМО с полными данными. Все данные сохранены в CSV. Главный урок — перед тем как писать код, стоит вручную изучить структуру страницы через инспектор браузера, это экономит время на отладку. Также важно использовать sleep между запросами, чтобы не нагружать сервер. Для парсинга нескольких страниц (например, main_news) удобно собирать ссылки в список, чтобы потом идти по ним отдельно и не пропускать дубликаты.
